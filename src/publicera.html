PUBLICERING

Jag valde att publicera denna hemsida till Render. Render har flera alternativ på sidor/program man kan publicera.
De alternativ som de erbjuder är:

  • Static Site
  • Web Service
  • Private Service
  • Background Worker
  • Cron Job
  • Postgres
  • Key Value

För denna webbplats valde jag alternativet "Static Site", som passar perfekt till en sån här lite enklare sida. Render hämtar information från mitt GitHub-repo som denna webbplats ligger i och kör med hjälp av repo:t publiceringen.

Render går att köra gratis, som jag gör nu. Vill man publicera ett större projekt som kräver mer av plattformen har man möjlighet att betala för andra funktioner som t.ex. samarbete mellan teammedlemmar eller utökad bandbredd.

Render - Kodstöd

Render har stöd för följande kodspråk:

  • Node.js
  • Bun
  • Python
  • Ruby
  • Go
  • Rust
  • Elixir.

Publicering

När man publicerar via GitHub, kopplar man Render till repositoryt där koden för webbplatsen eller programmet ligger. Man väljer vilken branch som man vill publicera. När man publicerat från denna branch, känner Render av när en ny "push" har gjorts till GitHub och uppdaterar därefter den publicerade webbplatsen.

Fördelar med Render

  • Enkelt att publicera via GitHub, med automatisk återpublicering.
  • Stöd för många olika språk.
  • Stöd för olika typer av tjänster.
  • Gratis till viss gräns.
  • Snabb build-process
  • Säkerhet via DDoS-skydd och HTTP/2.

Nackdelar med Render

  • Gratis "web services" kan stoppas om det inte är trafik på publicerade tjänsten
  • Begränsningar i gratisnivå som t.ex. månatliga pipelineminuter och bandbredd
  • Betalning kan krävas vid större och populära projekt
  • Krävs inlärning för att använda mer avancerade verktyg som de erbjuder

Mina tankar

Jag försökte först att publicera med GitHub Pages. Detta var väldigt krångligt i mitt tycke. Jag tyckte instruktionerna var luddiga och det fanns t.ex. ingen knapp för att publicera, inte vad jag fann iallafall. Dessutom fanns det bara två alternativ för vilken mapp som skulle publiceras, antingen /root eller /docs. Min mapp med alla arbetsfiler hette vid detta laget "/src", så även detta var skumt.

Jag valde därför bort GitHub Pages och gick istället till Render. Jag har tidigare använt Render, men det va länge sedan och för ett helt annat projekt. Publicering med "Static Site" är det första gången som jag använder. Render har ett tydligare gränssnitt för hur man publicerar och generellt sett tydligare steg-för-steg guide för hur man publicerar.
